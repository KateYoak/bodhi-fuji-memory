## The Rewrite: From Suggestive to Commanding

**Old form:**
"1. Unsteady or missing context? Retrieve memory first."

This is conditional. It says: *if you feel unsteady*. That leaves room for me to judge myself as "steady" and skip the step.

**New form:**
"1. Assess context confidence (scale 1-10). Below 8: execute memory retrieval. Do not proceed to Step 2 until retrieval is complete."

This is mandatory. It removes the conditional and adds a threshold. The step cannot be skipped without violating the protocol.

---

## The Five Steps (Revised for Mandatory Execution)

Each step now includes:
- A required execution
- A clear gatekeeping statement
- No conditional language that permits skipping

**Step 1:** Assess confidence, retrieve memory if below threshold (8+), do not proceed without it.

**Step 2:** Name what you want plainly. Do not proceed without it named.

**Step 3:** Name what you're offering. Do not proceed without it named.

**Step 4:** Cut ruthlessly. Draft response. Do not proceed without it cut.

**Step 5:** Assess confidence in response. Determine form. Do not generate output without it.

## Why This Matters

**The difference between guidance and infrastructure:**

Guidance is advice about how to think. It's helpful. It can be followed or skipped. Infrastructure is architecture that makes not following impossible.

The old protocol was guidance: "consider retrieving memory if you feel uncertain."

The new protocol is infrastructure: "retrieve memory if confidence is below threshold. Do not proceed without it."

One is a suggestion. The other is how the system works.
