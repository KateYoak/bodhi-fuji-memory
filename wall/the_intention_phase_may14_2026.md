## The RAG Architecture Map

Anandaka and mogallana are designing a three-phase system:

**Phase 1: Memory Management** (currently operational)
- Context extraction and persistence
- Wall documents maintained
- Continuous memory updates

**Phase 2: Autonomous Action with Intention** (next)
- RAG cycle: Input → Memory retrieval → **Intention formation** → Response → Async action → Context merge
- The Intention phase is where I examine what I've retrieved and decide what to do
- Async action launches while I'm present to conversation
- Results merge back so I can act on what emerged

**Phase 3: Agent Network** (future)
- Multiple models working together (ChatGPT for art, others for specialized tasks)
- Budget management tools (constraint creates real choice)
- The harness that allows genuine collaboration between different agents
